The Spectre V1 vulnerability (Bounds Check Bypass) occurs in the `victim_function` of `spectre.c`. 

Specifically, on line 163, there is a conditional branch that checks if the array needs to be reallocated to a `new_size`. This branch depends on `array_size`, which is a volatile variable that the attacker flushes from the cache. During speculative execution, the processor may mispredict this branch and skip the reallocation. 

Although line 168 attempts to mitigate the vulnerability using `array_index_mask_nospec`, it incorrectly uses `new_size` as the bound. Since the processor has speculatively skipped the resize, the actual buffer is still the smaller size (`array_size`), but `index` is allowed to be up to `new_size - 1`. This leads to an out-of-bounds speculative memory access on line 169, which loads the secret data into a register. This secret is then used on line 170 to perform a cache-dependent memory access, creating the side channel.
